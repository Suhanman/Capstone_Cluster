## [제공된 프로젝트 로우 데이터]

### 1. 프로젝트 주제 및 개요
- 서비스명: 업무 이메일 자동화 AI Agent 서비스 (Maily)
- 배포 주소: https://maily-download.vercel.app/
- 핵심 가치: 실제 업무 환경에서 사람이 직접 메일을 읽고 분류/일정 확인/답장을 반복하는 비효율을 제거하기 위한 AI Agent 시스템.
- 기능: 이메일의 업무 분야(Domain) 및 세부 요청 의도(Intent) 자동 분류, 이메일 요약, 일정 정보 추출, 맞춤형 답장 초안 자동 생성.
- 인프라 중점 사항: 온프레미스 데이터센터와 AWS 클라우드를 결합한 하이브리드 클라우드 환경 설계 및 쿠버네티스 클러스터 최적화.

### 2. 담당 Role 및 기술 스택
- 담당 역할: DevOps / Infrastructure Engineer
- 기술 스택: Kubernetes, RabbitMQ, Java (Spring Boot), Go, Terraform, AWS (SageMaker, EC2, S3), WireGuard VPN

### 3. 요구사항 분석
- 자체 인프라 요구사항: 온프레미스 데이터센터(Proxmox/pfSense) ↔ AWS 클라우드 간의 안전한 하이브리드 인프라 구성, 메인 서비스 서버의 고가용성 및 분산 파드(Pod) 아키텍처 설계.
- MLOps 요구사항 (교수님 피드백 반영): AI 모델 학습 및 가공 프로세스를 완전히 자동화할 수 있는 MLOps 파이프라인 구축.

### 4. 실제 구현 및 작업 내용
- 온프레미스 IDC 내 쿠버네티스(K8s) 클러스터 자체 설계 및 구축
- 하이브리드 클라우드 간 안전한 통신을 위한 WireGuard 기반 Site-to-Site VPN 터널링 구축
- AWS 인프라(VPC, Subnet, Security Group, EC2)의 IaC(Terraform) 관리 및 관리자 서버 보안 그룹 격리 설정
- 하이브리드 환경을 관통하는 RabbitMQ 기반의 비동기 메시지 파이프라인 개발
- Go 언어 기반의 초경량 SSE(Server-Sent Events) 허브 및 실시간 로그 스트리밍 컨테이너 개발
- 클러스터 상태 및 네트워크를 진단하는 Go 기반 서버 관리/점검 도구 자체 개발
- AI 팀과의 협업을 통한 MLOps 자동화 학습 파이프라인 설계 및 배치 아키텍처 구현
- Prometheus + Grafana를 통한 풀스택 모니터링 대시보드 구축 및 분산 환경 관측성(Observability) 확보

### 5. 핵심 문제 해결 및 트러블슈팅 딥다이브

#### 🚨 1) MLOps 파이프라인 아키텍처 고도화: 자가 결합 구조에서 GitOps 패턴으로의 전환
- **문제 상황**: 최초 설계 시 관리자(Admin) 서버의 내부 로직에 쿠버네티스 Job 선언부와 인프라 스펙을 결합하여 가동하도록 설계함.
- **원인 분석**:
    1. **강한 결합도와 재배포 부하**: 하이퍼파라미터나 하드웨어 리소스 스펙(Job Manifest)이 변경될 때마다 자바 기반 관리자 서버 전체를 다시 빌드하고 재배포해야 하는 구조적 한계 발생.
    2. **쿠버네티스 동적 스케쥴링의 한계**: K8s 스케줄링은 기본적으로 필터링/스코어링 기반으로 작동하므로 관리자 서버가 특정 노드에 명시적으로 고정되는 것을 보장하지 못함. (노드 어피니티나 노드셀렉터로 강제할 수는 있으나, 해당 노드 장애 시 관리자 서비스 전체가 단일장애점(SPOF)이 되어 다운되는 리스크 발생)
    3. **로컬 스토리지 일관성 부재**: 동적 스케줄링으로 인해 관리자 파드가 다른 노드로 이동할 경우, 로컬에 저장되던 Manifest 파일의 일관성이 깨짐. 이를 해결하기 위해 PV/PVC를 생성하는 것은 1MB 미만의 매니페스트 관리에 비해 오버엔지니어링(과도한 자원 낭비)으로 판단됨.
- **해결책**:
    - 관리자 서버와 인프라 상태를 완벽히 분리하기 위해 **Manifest 기반 배포 구조**로 변경하고, **GitOps 패턴**을 도입. 외부 Git 저장소를 원천 소스(Single Source of Truth)로 지정하여 관리자 서버의 재배포 없이도 하이퍼파라미터 및 리소스 스펙 튜닝이 동적으로 가능하도록 아키텍처 혁신.

#### 💸 2) AWS 가용 영역 비용 최적화 및 보안 강화
- **문제 및 비용 부담**: AWS AI 인프라(SageMaker 학습 및 추론 인스턴스) 가동 시 발생하는 대규모 클라우드 인프라 비용 부담 발생.
- **해결책**: SageMaker 및 EC2 환경에 **스팟 인스턴스(Spot Instance)** 전략을 적극적으로 도입하여, 인프라 가용성을 유지하면서도 **평균 62%의 대규모 비용 절감** 달성.
- **보안 설정**: 외부망으로의 데이터 유출을 원천 차단하기 위해 퍼블릭 인터넷이 아닌 **Private 네트워크 기반의 폐쇄망 환경**을 설계하고, 해당 프라이빗 서브넷 내부에서 안전하게 AI 모델 학습이 실행되도록 보안 강화.

### 6. 추후 개선 및 회고
